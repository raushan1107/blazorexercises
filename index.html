RR Skillverse – Blazor Server Hands-On Lab (.NET 6)

Lab Overview

In this lab, you will build a Helpdesk Portal using Blazor Server (.NET 6) and understand how real-time UI updates work.

Prerequisites

  • .NET 6 SDK installed
  • Visual Studio 2022 OR VS Code
  • Basic C# knowledge

Task 1 – Create Blazor Server Project

Step 1: Open Terminal

Open Command Prompt or Terminal.

Step 2: Create Project
dotnet new blazorserver -n HelpdeskPortal
Step 3: Navigate to Project
cd HelpdeskPortal
Step 4: Run Application
dotnet run
✔ You should see: “Now listening on https://localhost:xxxx”

Task 2 – Create Ticket Model

Step 1: Open Project in Visual Studio

Open the HelpdeskPortal folder.

Step 2: Create Folder

Right click project → Add → New Folder → Name it Models

Step 3: Create Class

Inside Models → Add → Class → Name it Ticket.cs

Step 4: Replace Code
public class Ticket
{
    public int Id { get; set; }
    public string Title { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
    public string CreatedBy { get; set; } = string.Empty;
    public bool IsResolved { get; set; }
    public DateTime CreatedAt { get; set; }
}
Model defines structure only. No business logic here.

Task 3 – Create Ticket Service

Step 1: Create Services Folder

Right click project → Add → New Folder → Name it Services

Step 2: Add TicketService.cs
Step 3: Paste Service Code
using HelpdeskPortal.Models;

public class TicketService
{
    private readonly List<Ticket> tickets = new();
    private int nextId = 1;

    public event Action? OnChange;

    public List<Ticket> GetTickets() => tickets;

    public void AddTicket(Ticket ticket)
    {
        ticket.Id = nextId++;
        ticket.CreatedAt = DateTime.Now;
        tickets.Add(ticket);
        NotifyStateChanged();
    }

    public void ResolveTicket(int id)
    {
        var ticket = tickets.FirstOrDefault(t => t.Id == id);
        if (ticket != null)
        {
            ticket.IsResolved = true;
            NotifyStateChanged();
        }
    }

    private void NotifyStateChanged() => OnChange?.Invoke();
}

Task 4 – Register Service

Step 1: Open Program.cs
Step 2: Add This Line
builder.Services.AddSingleton<TicketService>();
Singleton shares data across all connected users.

Task 5 – Verify Real-Time Behavior

  • Run application
  • Open two browser windows
  • Create ticket in one
  • Observe other browser updating instantly
✔ You should see automatic UI refresh without reloading page.
